# メンバーリストマネージャー

Google Apps Script (GAS) を用いて, LINE Webhook からのメッセージに応じて会員名簿スプレッドシートを更新し, LINEへ自動返信するシステムである.

## ディレクトリ構造
- config.js : 定数および設定定義ファイル.
- log.js : 実行ログ記録およびスプレッドシート一括書き込み処理.
- process.js : スプレッドシート更新（昇級, 区分変更, 休退会, 入会）処理ロジック.
- webhook.js : LINE Webhook の受信およびコマンド解析, LINE返信処理.
- test.js : モックを用いたユニットテスト実行コード.
- package.json : npm 環境定義ファイル.

## 開発知見および注意点

### 1. LINE Webhook における返信仕様の注意
LINE API を使用して Webhook に返信（Reply）を行う場合, API の宛先パラメータには event.source.userId 等のユーザーIDではなく, event.replyToken を指定する必要がある. ユーザーIDはプッシュメッセージ（Push）用であるため, リプライ用のエンドポイントに渡すとエラーとなる.

### 2. PropertiesService を用いたアクセストークン管理
アクセストークンなどの機密情報はコード上に直接記述せず, GAS の ScriptProperties に設定する設計を優先する. プロパティが存在しない場合は config.js のフォールバック値を使用する構成にすることで, 安全性とポータビリティを両立している.

### 3. WebApp 実行時のデバッグログ機能
GASの WebApp (doPost) 実行時の標準ログは, GASエディタの「実行数」に表示されない.
この問題を解決するため, 本システムはスプレッドシート内に「実行ログ」シートを自動生成し, そこへ時系列でログを出力する.
- パフォーマンスへの影響を最小限にするため, ログはメモリに蓄積し, リクエスト終了時に一括して書き出すバッチ処理を採用している.
- シートが重くなるのを避けるため, ログが1000行を超えると古いログ行が自動的に削除される.

### 4. LINE コマンドにおける日付指定と自動補完仕様
すべての LINE コマンドにおいて「日付：yyyy/mm/dd」形式で処理適用日を指定できる.
- 指定可能な形式: yyyy/mm/dd, mm/dd, m/d (スラッシュまたはハイフン区切り).
- 西暦 (yyyy) が不足している場合, 基準日（本日）に対して「前年」「今年」「来年」の同月日のうち, 日数差が最も小さい（今日に最も近い）日付を自動で算出して補完する.
  - 例: 2025/01/02 に「日付：12/20」を受信した場合, 最も近い過去の日付である 2024/12/20 と判定する.
  - 例: 2026/05/25 に「日付：6/1」を受信した場合, 2026/06/01 と判定する.

### 5. GAS 環境のローカルモックテスト
GAS 上で動作するコードはグローバルオブジェクト（SpreadsheetApp, UrlFetchApp, Utilities等）に依存するため, ローカルの Node.js 環境では動作しない. 
本プロジェクトでは test.js 内の setupMocks 関数でこれらを一時的にモック化する手法を採用しており, 以下のコマンドによりローカル Node.js 環境下でも完全にテストを実行できる.

コマンド:
npm test

### 6. GAS へのデプロイ
ユーザーが手動で clasp push を実行可能にするため, npm スクリプトとして clasp push を設定している.

コマンド:
npm run push

### 7. 既存メンバー向けコマンドのフルネーム入力と自動解決仕様
既存メンバーを対象とするコマンド（★昇級★, ★区分変更★, ★休退会★）では, 従来の姓名の個別入力（姓：, 名：）に加え, フルネーム（姓名：, 氏名：, 名前：, フルネーム：）による入力も受け付ける.
- 入力されたフルネーム値（および「姓」＋「名」から組み立てられた値）から, すべてのスペース（半角および全角）を除去した検索用のフルネームを作成する.
- スプレッドシートの「全会員名簿」内の全会員データから, 「姓」と「名」を結合してスペースを除去した文字列が, 検索用フルネームと完全一致するレコードを走査する.
- 一致するレコードが見つかった場合, そのレコードの「姓」および「名」へ自動的に解決する. 見つからない場合はエラーメッセージをLINEで返信する.
- 新規入会および復帰（★入会★, ★復帰★）コマンドについては, スプレッドシートに既存レコードがないため本解決ロジックは適用されず, 従来通り「姓」および「名」の個別入力を必須とする.
