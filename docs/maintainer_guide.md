# 名簿マネージャー メンテナンスガイド

このドキュメントは, 名簿マネージャー（master）および名簿ビューア（viewer）のシステム管理およびメンテナンスを行う担当者向けの手引書である.

---

## 1. システム全体の仕組み
本システムは, LINE Botを介して名簿データを更新するマスターシステムと, そのデータを事務作業用に閲覧・コピーするためのビューアシステムの2つで構成されている.

[LINE ユーザー]
    | (メッセージ送信: 例: ★昇級★ フルネーム：山田太郎 新級：A)
    v
[LINE サーバー]
    | (Webhook 送信)
    v
[Google Apps Script (master)]
    | (スプレッドシートの更新および「実行ログ」への書き込み)
    v
[スプレッドシート (マスター名簿)] <--- (毎日深夜に自動同期) ---> [Google Apps Script (viewer)]
                                                                    |
                                                                    v
                                                            [コピペ用データ生成]
                                                                    |
                                                                    v
                                                            [スプレッドシート (ビューア名簿)]

---

## 2. 必要な環境とツール
システムの管理には以下のツールを使用する.
- Google Apps Script (GAS)
- LINE Developers コンソール
- Node.js (任意, claspを利用したローカル開発を行う場合)

---

## 3. master（名簿更新システム）の初期設定とデプロイ

### 3.1. スプレッドシートの作成
1. マスター用の Google スプレッドシートを新規作成する.
2. スプレッドシート内に以下のシートを作成する.
   - 「全会員名簿」
   - 「正准会員名簿」
   - 「昇級履歴」
   - 「ステータス履歴」
   - 「実行ログ」
3. 各シートの1行目に, src/config.js の MEMBER_HEADER 等で定義されているヘッダー名を記述する.
4. 作成したスプレッドシートのID（URLの /d/ と /edit の間の文字列）をメモしておく.

### 3.2. コードのアップロード
claspを利用してアップロードする場合の手順は以下の通りである.
1. `master` ディレクトリでターミナルを開く.
2. `npm install` を実行して依存パッケージをインストールする.
3. `.clasp.json` を作成し, `scriptId` に対象の GAS スクリプト ID を設定する. また, `"rootDir": "src"` を指定して src ディレクトリ配下のファイルのみがアップロードされるように設定する.
4. `npm run push` コマンドを実行してコードをアップロードする.

### 3.3. スクリプトプロパティの設定
1. GASエディタの画面左側メニューにある「プロジェクトの設定」（歯車アイコン）を開く.
2. 「スクリプトプロパティ」セクションで「スクリプトプロパティを追加」をクリックする.
3. 以下のプロパティを追加して保存する.
   - プロパティ名: LINE_TOKEN
   - 値: LINE Developers で取得した「チャネルアクセストークン」

### 3.4. Webアプリとしてのデプロイ
1. GASエディタの画面右上にある「デプロイ」 > 「新しいデプロイ」をクリックする.
2. 種類の選択で「ウェブアプリ」を選択する.
3. 以下の設定項目を入力する.
   - 次のユーザーとして実行: 「自分」（管理者アカウント）
   - アクセスできるユーザー: 「全員」
4. 「デプロイ」を実行する.
5. 発行された「ウェブアプリのURL」をコピーして控えておく.

### 3.5. LINE Webhook の設定
1. LINE Developers コンソールにログインする.
2. 対象のプロバイダーおよびチャネルを選択する.
3. 「Messaging API設定」タブを開く.
4. 「Webhook URL」の項目に, 先ほどコピーした「ウェブアプリのURL」を貼り付ける.
5. 「Webhookの利用」を有効（ON）にする.

## 4. viewer（名簿ビューア）の初期設定とデプロイ

### 4.1. スプレッドシートの作成
1. ビューア用の Google スプレッドシートを新規作成する.
2. 「拡張機能」メニューから「Apps Script」を開く.
3. 作成したスプレッドシートのIDをメモしておく.

### 4.2. config.js の設定
1. `viewer/src/config.js` を開く.
2. `SPREADSHEET_ID` 定数の値を, 3.1. で控えた「マスター用のGoogleスプレッドシートID」に変更する.

### 4.3. コードのアップロード
1. `viewer` ディレクトリでターミナルを開く.
2. `npm install` を実行して依存パッケージをインストールする.
3. `.clasp.json` を作成し, `scriptId` にビューアの GAS スクリプト ID を設定する. また, `"rootDir": "src"` を指定して src ディレクトリ配下のファイルのみがアップロードされるように設定する.
4. `npm run push` コマンドを実行してコードをアップロードする.

### 4.4. 自動同期トリガーの設定
1. ビューア用スプレッドシートを開く.
2. スプレッドシートのメニューに「名簿ビューア」が表示されていることを確認する.
3. 「名簿ビューア」 > 「【管理者用】自動同期トリガーを設定」を実行する.
4. 初回実行時は Google アカウントによる認証認可の画面が表示されるため, 画面の指示に従って承認する.
5. これにより, 毎日深夜（午前4時〜5時の間）にマスターからデータを自動取得し, コピペ用のプレビューを更新するトリガーが設定される.

## 5. 日常のメンテナンスとトラブルシューティング

### 5.1. LINEメッセージを送っても名簿が更新されない場合
1. LINE Developers コンソールの「Messaging API設定」タブにある Webhook URL が正しいか確認する.
2. マスター用スプレッドシートの「実行ログ」シートを確認する. 処理エラーや例外が記録されている場合はその内容を確認する.
3. LINEのチャネルアクセストークンの有効期限が切れていないか確認する.
4. GASエディタの「実行数」メニューから doPost 関数のエラー履歴を確認する.

### 5.2. ビューアのデータが更新されない場合
1. ビューア用スプレッドシートのメニュー「名簿ビューア」 > 「【緊急用】マスターから手動同期」を実行し, 即時同期できるか確認する.
2. ビューア用スプレッドシートの Apps Script エディタを開き, 左側の「実行数」メニューから `syncFromMaster` がエラーで終了していないか確認する.
3. マスター用スプレッドシートID（`SPREADSHEET_ID`）が `viewer/src/config.js` に正しく記述されているか確認する.
4. ビューア用スプレッドシート内に必要なシート（「名簿一覧」, 「コピペ用名簿」, 「ソート名簿コピペ」）が揃っているか確認する（存在しない場合は初期化処理により自動作成される）.
