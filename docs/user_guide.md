# 名簿マネージャー 利用ガイド

本ドキュメントは, LINEを活用した会員名簿の自動更新システムおよび閲覧用ビューアの全体像, 導入メリット, ならびに日常の利用手順について説明する.

---

## 1. システム導入の背景と目的
従来の名簿管理業務では, 会員の昇級や区分の変更が発生するたびに, 担当者が手動でスプレッドシートを更新し, さらに連絡用の名簿テキストを手動で作成してコピペしていた.
このため, 以下の課題が存在していた.
- 手動更新による入力ミスや表記ゆれの発生.
- 名簿コピペ用テキストを作成する際の手間の発生とフォーマットミスのリスク.

本システムの導入により, 以下の目的を達成する.
- LINEからの簡単なメッセージ送信による名簿データの安全かつ迅速な自動更新.
- コピペ用の名簿プレビューテキストの自動生成による事務作業の効率化とミス防止.

---

## 2. システムの全体像とメリット
本システムは, LINE Botを介して名簿データを更新するマスターシステムと, そのデータを事務作業用に閲覧およびコピーするためのビューアシステムの2つで構成されている.

[LINEからの簡単なコマンド送信] ---> [マスター名簿の自動更新] ---> [毎日深夜の自動同期] ---> [ビューア名簿での自動プレビュー生成] ---> [ワンクリックでコピー完了]

### システム導入によるメリット
1. <b>ミス削減</b>: データの更新はLINEコマンドで行い, スプレッドシートの関数や手動操作によるデータの破壊を防ぐ.
3. <b>操作の柔軟性</b>: スプレッドシート上で標準のフィルター機能を用いて並び替えた順のままコピーすることも可能である.

---

## 3. 実務担当者の日常の操作手順

### 3.1. ビューア名簿からのコピー手順
1. ビューア用スプレッドシート（閲覧用）を開く.
2. 拡張機能メニューの隣にある「名簿ビューア」 > 「サイドバーを表示」をクリックする.
3. 画面の右側に操作用のサイドバーが表示される.
4. サイドバーに表示されている以下のボタンをクリックすることで, クリップボードに名簿テキストがコピーされる.
   - 「LINE更新用名簿をコピー」: タイトルや注意書きを含めたLINE連絡用の形式でコピーする.
   - 「正准会員名簿をコピー」: 級（A級から入門級）でグループ化し, ふりがな順に並べ替えた形式でコピーする. 准会員は学年の後ろに「(准)」が付き, 基本および入門級は入会月が自動付与される.
   - 「メイト会員名簿をコピー」: ふりがな順に並べ替えた, 姓名とふりがなのみのフラットな形式でコピーする.
5. コピーされたテキストを, LINEやメールなどの連絡ツールにそのまま貼り付けて使用する.

### 3.2. 任意のソート順でコピーする手順
1. ビューア用スプレッドシートの「ソート名簿コピペ」シートを開く.
2. スプレッドシートの標準フィルター機能等を利用し, データを任意の順番（例: ふりがな順, 会員区分順など）に並び替える.
3. サイドバーの「現在のソート順でコピー」ボタンをクリックする.
4. 画面に表示されている通りの並び順で名簿テキストがクリップボードにコピーされる.

### 3.3. 手動での即時同期手順
通常, データは毎日深夜にマスター名簿から自動で同期されるため手動での操作は不要である.
日中などにLINEで行った変更を即座にビューアに反映させたい場合は, 以下の手順を実行する.
1. ビューア用スプレッドシートのメニューから「名簿ビューア」 > 「【緊急用】マスターから手動同期」をクリックする.
2. 画面右下に「マスター名簿からの同期が完了しました」とトースト通知が表示されたら, 同期完了である.

---

## 4. LINEからの名簿更新コマンドについて
LINE Botへメッセージを送ることで, マスター名簿のスプレッドシートを更新できる.
メッセージは必ず「★コマンド名★」から開始し, 各項目をコロン「：」または「:」で区切って記述する.
項目名やコロンは半角と全角のどちらでも認識される.

### 4.1. 昇級処理
会員が昇級した際に使用する.
```
★昇級★
フルネーム：山田太郎
新級：A
日付：12/20
```
- 新級に指定できる値: A, B, C, D, E, F, G, 基本, 入門
- 日付は任意である. 省略した場合は「本日」が適用される.
- 既存メンバーを対象とするため, 「フルネーム」を入力することで自動的に対象メンバーが検索される.

### 4.2. 会員区分変更
会員のステータス（正会員, 准会員, メイト会員）を変更する際に使用する.
```
★区分変更★
フルネーム：山田太郎
新区分：准会員
日付：12/20
```
- 新区分に指定できる値: 正会員, 准会員, メイト会員
- 日付は任意である. 省略した場合は「本日」が適用される.

### 4.3. 休退会処理
会員が休退会する際に使用する.
```
★休退会★
フルネーム：山田太郎
日付：12/20
```
- 会員区分が「休退会」へ変更され, 日付が適用日として記録される.
- 日付は任意である. 省略した場合は「本日」が適用される.

### 4.4. 新規入会および復帰
新しく会員を登録する際, または休退会から復帰する際に使用する.
既存レコードが存在しないため, 姓名やふりがなを個別に指定する.
```
★入会★
姓：山田
名：太郎
姓ふりがな：やまだ
名ふりがな：たろう
学年：小1
級：入門
区分：正会員
日付：12/20
```
- 新規入会および復帰の場合はすべての項目が必須である.
- 日付を省略した場合は本日が入会日として登録される.
- 学年欄に半角数字・全角数字・漢数字が入力された場合（例: 小1, 小１, 小一）, 表記揺れを防ぐためシステムにより自動的に半角数字（例: 小1）へ正規化される.

### 4.5. 日付の指定と自動補完仕様について
「日付：」の項目では, 年（yyyy）を省略して「12/20」や「5/5」のように月日だけで指定可能である.
この場合, システムは「本日の日付」を基準として, 「前年」「今年」「来年」の同じ月日のうち, 最も本日と日数差が近い日付を自動で算出して補完する.
- 例: 本日が 2026/05/25 のときに「日付：6/1」と送ると, 最も近い 2026/06/01 に補完される.
- 例: 本日が 2026/05/25 のときに「日付：12/20」と送ると, 最も近い 2025/12/20 に補完される.
意図しない日付の補完を防ぐため, 過去または未来に大きく離れた日付を指定する場合は「yyyy/mm/dd」と西暦を含めて入力することを推奨する.
