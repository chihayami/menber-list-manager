# メンバーリストビューア (viewer)

名簿管理システム (master) のスプレッドシートからデータを取得し, 事務作業で参照・コピペするためのビューアシステムである.
本システムはトリガーベースの自動同期およびユーザーによる誤操作を防ぐための最小限のGUI設計を採用している.

## ディレクトリ構造
- config.js : 定数および設定定義ファイル.
- sync.js : masterからのデータ同期ロジックおよび自動同期トリガー管理.
- generator.js : コピペ用テキストの生成ロジック.
- コード.js : スプレッドシートのメニュー追加やシートの初期化などの制御フロー.
- clipboard.html : クリップボードへコピーするためのモーダルダイアログ用HTML.
- sidebar.html : 操作パネルとなるサイドバーUI用HTML.
- test.js : ローカル環境で実行するユニットテストコード.
- package.json : npm環境定義ファイル.

## 主要機能

### 1. マスターからの自動同期
- 日常の同期作業は全自動化されている. 管理者が自動同期トリガーを設定することで, 毎日深夜（午前4時〜5時の間）にマスター名簿から最新のデータを自動的に同期する.
- 同期処理の実行時に, 自動的にデータを「学年順」でソートする.
- 同期完了時に自動でスプレッドシートの標準フィルターを有効化するため, GUIを使った絞り込みがそのまま利用できる.
- 緊急で手動同期が必要な場合は, メニュー「名簿ビューア > 【緊急用】マスターから手動同期」または右側サイドバーから即座に実行可能である.

### 2. コピペ用名簿のプレビューとコピー
- コピペ用名簿シートには, 最新 of 同期データに基づくプレビューテキストが自動で常時書き出されている（ボタン操作やセル選択による手動更新は不要）.
  - A列: LINE更新用名簿のプレビュー（2行目）
  - B列: 正准会員名簿のプレビュー（2行目）
  - C列: メイト会員名簿のプレビュー（2行目）
- ユーザーはプレビューセルを直接選択してコピーするか, 画面右側のサイドバーに表示されるコピーボタンをクリックすることで, クリップボードにテキストをコピーできる.
- それぞれの出力形式は以下の通りである.
  - LINE更新用名簿:
    タイトル（ちはやふる富士見 会員名簿）, 更新日（yyyy/mm/dd）, 持ち出し禁止の注意書きをヘッダーとし, その後に正准会員およびメイト会員のコピペ用名簿テキストを結合して出力する.
  - 正准会員名簿:
    級（A級から入門級まで）にグループ化し, ふりがな五十音順に並べ替えて出力する. 姓名の間のスペースは排除され, 姓名, ひらがな, 学年の間はタブ文字「\t」で区切られる. 准会員は学年の後ろに「(准)」が付き, 基本・入門級はさらに後ろに入会月が [yymm] 形式で自動付与される.
  - メイト会員名簿:
    級によるグループ化や学年の出力は行わず, 全員をふりがな順に並べて姓名とふりがなのみ（例: 佐藤メイト\tさとうめいと）をフラットに出力する.

### 3. ソート順でのコピー
- ソート名簿コピペシート上で標準フィルター等を使って任意の順番に並べ替えた状態で, サイドバーの「現在のソート順でコピー」をクリックすると, 画面に表示されている順番通りの名簿テキストがコピーされる.

## 初期設定手順
1. claspをセットアップし, viewerディレクトリ配下で以下のコマンドを実行してコードをGoogle Apps Scriptにアップロードする.
   npm run push
2. 紐づくスプレッドシートを開く. 必要なシート（名簿一覧, コピペ用名簿, ソート名簿コピペ）が存在しない場合は, 自動的に初期化処理が走りシートが作成される.
3. メニュー「名簿ビューア > 【管理者用】自動同期トリガーを設定」を実行する. これにより、毎日深夜の自動同期処理が有効化される.
4. 初回実行時や即座にデータを反映したい場合は, メニューから「【緊急用】マスターから手動同期」を実行する.

## 開発知見および技術的注意点

### 1. クリップボードへの自動コピー制限の回避
Google Apps Scriptのサーバーサイドコードからは, クライアント（ローカルPC）のクリップボードへ直接テキストを書き込めない.
これを解決するため, 本システムでは以下の2つのアプローチを提供している.
- サイドバー式: サイドバーUI（HTML）上からクライアントサイドJavaScript（navigator.clipboard.writeText）を呼び出し, 直接コピーを行う.
- メニュー式: メニューから実行した際, 極めて軽量な一時モーダルダイアログ（clipboard.html）を一瞬開き, クライアントサイドのJavaScriptでコピーを実行させると同時に自身を閉じる処理（google.script.host.close）を走らせている.

### 2. 擬似セルボタン廃止とトリガー自動同期への移行経緯
かつてはシートの1行目に「擬似セルボタン」を設置し, `onSelectionChange(e)` トリガーでプレビューを更新する設計を採用していた. しかし, このアプローチには以下の課題があった.
- セルを選択するだけで重い生成処理が走るため, ユーザーの誤操作による意図しない実行やGASの過剰な実行枠消費が発生しやすかった.
- シンプルトリガーの制限により, セルクリックから直接コピーダイアログを表示することができず, 操作が煩雑になっていた.

これらを根本的に解決するため, 本設計では「擬似セルボタン」と `onSelectionChange` トリガーを完全に撤去した. 代わりに, 深夜の「時間主導型トリガー」による自動同期の末尾で, あらかじめプレビューテキストを自動生成してセルに書き込んでおく設計へと移行した. これにより, ユーザーはシートを開いた時点で即座に最新のテキストをコピーできるようになり, 処理効率と操作性が大幅に向上した.

### 3. データの書き込み・読み込みインデックス
操作用擬似ボタン行が廃止されたことに伴い, 各シートのデータ構造は以下のように極めてシンプルに統一された.
- 1行目: テーブルヘッダー（またはプレビューヘッダーラベル）
- 2行目以降: データ行（またはプレビュー値）
これに伴い, 同期処理でのクリア・書き込みは1行目からを対象とし, コピペジェネレータでの走査ループはインデックス 1（2行目）から開始するシンプルな構造にリファクタリングされた.

### 4. スプレッドシートやUI操作に必要なOAuthスコープの明示
masterスプレッドシートを `SpreadsheetApp.openById()` で開いてデータを同期する処理や, サイドバーおよびモーダルダイアログを表示する処理を実行するには, 適切なOAuthスコープが必要である.
自動検出のみでは権限エラーが発生することがあるため, appsscript.json の oauthScopes に以下を明示的に設定してデプロイしている.
- https://www.googleapis.com/auth/spreadsheets
- https://www.googleapis.com/auth/script.container.ui

### 5. プレビューエラーの修正とテスト整合性の確保
名簿プレビュー処理において、変数の宣言順序に起因する不具合と、出力フォーマットの不一致が検出されたため、以下の修正を行った.
- 変数の初期化順序の修正:
  generateClassListText 関数内において、displaySchoolYear 変数の初期化前に変数への代入が行われていたため、初期化前にアクセスされたことによる ReferenceError が発生していた. 変数宣言を代入処理の前に移動することで解決した.
- 入会月フォーマットの統一:
  基本・入門級のメンバーに付与される入会月情報の囲み文字を、仕様に基づき大括弧 [ ] に統一した. これに合わせ、generateSortedListText の出力フォーマットおよびテストの期待値についても大括弧 [ ] を使用するように修正した.
- テストとのテキスト整合性:
  generateLineUpdateListText 関数の出力するテキストと、テストコード内のアサーション期待値に乖離（注意書きや見出しの有無）があったため、テストが期待するフォーマットに合わせて実装を修正した.
